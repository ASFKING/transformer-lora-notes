# 阶段 3：多头注意力与位置编码

> 状态：✅ 已完成 | 日期：2026-06-11

---

## 1. 单头注意力的局限

一个头 = 一套 $W_Q, W_K, W_V$ = 一种"看世界的方式"。

问题：一个头只能产出一套注意力权重，必须在多种关系之间"妥协"。

类比：一副眼镜只能有一种镜片，看远处的山和近处的书没法同时看清。

## 2. 多头注意力的结构

把 $W_Q, W_K, W_V$ 拆成 $h$ 个小矩阵，每个头独立计算注意力，最后拼接回来。

### 关键维度（以 d_model=8, h=2 为例）

| 矩阵 | 单头 (h=1) | 双头 (h=2) |
|------|-----------|-----------|
| 输入 X | 3×8 | 3×8 |
| W_Q | 8×8 | Head1: 8×4, Head2: 8×4 |
| W_K | 8×8 | Head1: 8×4, Head2: 8×4 |
| W_V | 8×8 | Head1: 8×4, Head2: 8×4 |
| Q, K, V | 各 3×8 | 各头 3×4 |
| head 输出 | 3×8 | 各头 3×4 |
| 拼接后 | 不需要 | 3×8 |
| W^O | 不需要 | 8×8 |
| **最终输出** | **3×8** | **3×8** |

核心规律：
- 词的维度始终是 d_model，输入输出不变
- 拆分的是 W_Q, W_K, W_V，把 d_model×d_model 拆成 h 个 d_model×(d_model/h)
- 每个头的输出维度 = d_model/h，拼接后恢复为 d_model
- 最终输出维度和输入维度相同，可以无限堆叠

### 拼接 = 按列拼

不是矩阵乘法，是物理拼接。每个词的向量按维度方向首尾相接：

```
head1: 3×4    head2: 3×4
  ↓              ↓
[head1[i] | head2[i]]  →  每个词从 4 维变 8 维
  ↓
Concat: 3×8
```

### W^O 的作用

拼接后乘以 W^O (d_model × d_model)，让不同头的信息互相融合。

拼接 = 把 8 份报告订在一起。
W^O = 总编辑，把 8 份报告综合成一份最终报告。

没有 W^O，各头的表示空间是孤立的，信息没有交叉。

## 3. 为什么需要位置编码

Transformer 是"圆桌会议"——所有词同时互相看，本身不知道谁在前谁在后。

如果没有位置编码：
- "我吃苹果" 和 "苹果吃我" 对 Transformer 来说完全一样
- 三个词的 Q、K、V 计算结果完全相同
- 注意力权重矩阵也完全相同

必须注入位置信息。

## 4. 位置编码的直觉

给每个位置一个独特的"签名"，加到词向量上：

输入 = Embedding(词) + PositionalEncoding(位置)

类比：一排人穿同样的白衣服，给每个座位贴不同颜色的标签来区分位置。

## 5. Sinusoidal 位置编码（绝对位置）

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$
$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

直觉：
- 每个位置得到一个唯一的向量（指纹）
- 不同频率的 sin/cos 组合，能生成足够多的唯一模式
- 高频（波长短）→ 区分相邻位置
- 低频（波长长）→ 区分远距离位置

类比：时钟的秒针（快）、分针（慢）、时针（更慢），三个指针组合唯一确定时间。

### 巧妙性质：相对位置可线性变换

$$PE_{pos+k} = T_k \cdot PE_{pos}$$

位置 pos+k 的编码可以由位置 pos 的编码通过固定线性变换得到。
模型可以泛化到未见过的长度。

## 6. 现代方案：RoPE（旋转位置编码）

不加到 Embedding 上，而是在计算 Q 和 K 的点积时，把位置信息"旋转"进去。

直觉：想象 Q 和 K 都是二维向量，RoPE 把每个位置的向量旋转一个角度。
两个向量的点积只取决于它们的夹角。旋转后点积只和相对距离 m-n 有关。

类比：两个人站在旋转木马上，从外面看他们的相对角度始终不变。

### RoPE 的优势

- 天然编码相对位置（不需要模型"学"相对关系）
- 长度外推性更好（处理比训练时更长的文本）
- Llama、Qwen、Mistral 等现代模型都用 RoPE

## 7. 绝对 vs 相对位置编码对比

| 特性 | Sinusoidal（绝对） | RoPE（相对） |
|------|-------------------|-------------|
| 注入方式 | 加到 Embedding 上 | 旋转 Q、K |
| 编码的是 | 绝对位置 | 相对距离 |
| 长度外推 | 较差 | 较好（配合 scaling 技术） |
| 代表模型 | 原始 Transformer、BERT | Llama、Qwen、Mistral |

---

## 核心要点

1. 多头 = 把 d_model 维度切成 h 份，每份独立算注意力，拼回来用 W^O 融合
2. 拼接是物理拼接（按列），不是矩阵乘法
3. W^O 让不同头的信息互相渗透，不是多余的
4. 没有位置编码，"我吃苹果"和"苹果吃我"一样
5. Sinusoidal 用不同频率的 sin/cos 唯一标识位置
6. RoPE 旋转 Q、K，天然编码相对位置，是现代 LLM 的主流方案

| 概念 | 类比 |
|------|------|
| 单头 vs 多头 | 一副眼镜 vs 多副眼镜 |

| W^O | 总编辑，综合 8 份报告 |
| Sinusoidal | 时钟的秒针/分针/时针 |
| RoPE | 旋转木马上两人的相对角度 |
| 无位置编码 | 一模一样的三个人，分不清谁是谁 |
