## 阶段 1 ✅ — 2026-06-02

### 核心收获
- RNN 的痛点：串行处理 + 长距离遗忘
- Transformer 的核心：全局扫描，所有词同时互相看
- Encoder-Decoder 架构：编码器（理解）+ 解码器（生成）
- 自注意力的直觉：每个词根据上下文重新定义自己的含义
- 自注意力对向量的操作：加权求和（线性组合）
- 完整流程：分词 → Embedding → N 层 Transformer Block → 取最后词向量 → 预测下一个 token
- 每一层 = 自注意力 + FFN，堆叠后逐层提炼语义
- 最后一个词为什么信息最全：因果注意力的单向汇聚

### 关键问答
- **为什么每个 token 要和其他 token 做查询？** 不是"绑定"最相似的词，而是决定"我该听谁的话、听多少"。通过加权混合，让每个词的向量融合上下文信息。
- **自注意力对向量做了什么？** 加权求和。新向量 = 所有词的向量按相关度比例混合。
- **为什么最后一个词包含完整上下文？** 因果注意力规则（只能看前面），信息像接力赛从左到右汇聚。
- **每一层是什么？** Transformer Block = 自注意力子层 + FFN 子层，堆叠 N 次逐步提炼语义。

## 阶段 2 ✅ — 2026-06-02（复盘完成：2026-06-11）

### 核心收获
- "图书馆找书"类比：Q = 我的需求，K = 书的标签，V = 书的内容
- Q、K、V 是同一个输入 X 分别乘以 W_Q、W_K、W_V 得到的三种投影
- W_Q、W_K、W_V 是模型训练出来的固定参数，所有输入共享
- 点积计算匹配度 → Softmax 归一化 → 加权混合 V → 得到新向量
- 点积大 = 向量方向接近 = 相关度高（几何直觉：a·b = |a|·|b|·cosθ）
- 除以 √d_k 的作用：防止 Softmax 梯度消失（缩放数值，避免 one-hot 极端分布）
- d_k = 向量维度，k 来自 Key

### 延伸讨论
- 每层结构相同但参数不同：底层学语法，中层学语义，高层学抽象理解
- 预测只用最后一个词的向量（因果注意力下信息最全）
- 解码过程：最终向量 → 线性层 → 50000 个候选词打分 → Softmax → 选最高分
- 词表里每个词只有一个固定向量，语义区分靠 Transformer 层的上下文加工
- 维度 = 特征数量，4096 维能编码丰富的语义信息
- "苹果"水果 vs 品牌：初始向量相同，经不同上下文的 Transformer 层加工后，最终向量不同
- 信息压缩与丢失：最后一个向量不可能无损包含所有信息，模型学会保留"预测下一个词"最需要的信息
- FFN 的作用：自注意力 = 词与词交流；FFN = 每个词独立消化，内化为更高层抽象
- 因果注意力的掩码机制：处理第 N 个词时只能看前 N-1 个词，通过掩码矩阵（0 保留，-∞ 屏蔽）实现

### 动手任务
- 实现了完整的 Self-Attention Python 脚本（attention_demo.py）
- 覆盖：点积 → 缩放 → Softmax → 加权求和的全流程
- 验证了"吃"经过注意力机制后的新向量计算

### 关键公式
```
Q = X · W_Q    （我想问什么）
K = X · W_K    （我的线索是什么）
V = X · W_V    （我的实际内容）

scores = Q · K^T          （点积算匹配度）
scaled = scores / √d_k    （缩放防爆炸）
weights = softmax(scaled)  （变成概率权重）
output = weights · V       （加权混合内容）
```

### 已确认掌握 ✅

- Q6: √d_k 缩放 — 防止点积值过大导致 Softmax one-hot，梯度只流过一个位置

### 复盘通过 ✅
- **Q4: W_Q 和 W_K 为什么要分开** ✅
  - 点积满足交换律：a·b = b·a
  - 若 W_Q = W_K → Q = K → scores[i][j] = Q_i·Q_j = Q_j·Q_i = scores[j][i] → 对称矩阵
  - 对称意味着"词i对词j的关注度 = 词j对词i的关注度"，不合理（"我爱你"≠"你爱我"）
  - 核心：需求（Query）和供给（Key）是两种不同的语义角色，必须分开投影
- **Q5: 点积衡量相关的语义直觉** ✅
  - W_Q 空间 = "我在找什么"（需求）
  - W_K 空间 = "我能提供什么"（供给）
  - 点积衡量的是需求与供给的匹配度，不是词与词的相似度
  - 语义相关的词之所以点积大，是因为模型训练后让需求和供给方向对齐
- **Q7: demo 里点积为 0 的原因** ✅
  - Q 和 K 是手工编造的，不是模型训练出来的
  - 模型训练会让 W_Q/W_K 投影出"语义相关的词方向对齐"的空间
  - 手工值没有经过训练，方向随机，点积为 0 是巧合
- **类比复述** ✅（导师修正版）
  - RNN vs Transformer：排队传话（只能听前一个人，信息衰减）vs 圆桌会议（所有人同时互相交流，无传递损耗）
  - W_Q/W_K/W_V：同一个人的三张面孔——需求（我要什么）、供给（我提供什么）、内容（我的真实值）
  - 因果掩码：考试不能翻到后面看答案。是规则而非训练工具，训练和推理时都存在，贯穿始终

## 🔶 阶段 3 — 2026-06-11（进行中）

### 已理解 ✅
- **多头注意力的核心价值**：不是保留维度，而是每个头专注学一种注意力模式（语法、语义、局部、长距离等），互不干扰
- **类比**：多色荧光笔——每种颜色标记不同关系，叠加后理解更立体
- **数学流程**：h 个头各自独立的 W_Q/W_K/W_V → 各自做 Attention → Concat → ×W_O → 输出
- **位置编码的必要性**：注意力公式中没有位置信息，"我吃苹果"和"苹果吃我"会被视为相同输入（bag of words）
- **绝对位置编码**：正弦/余弦生成唯一位置向量，加到词向量上（加法 = 特征融合，不改维度）
