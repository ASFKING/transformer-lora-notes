## 阶段 1 ✅ — 2026-06-02

### 核心收获
- RNN 的痛点：串行处理 + 长距离遗忘
- Transformer 的核心：全局扫描，所有词同时互相看
- Encoder-Decoder 架构：编码器（理解）+ 解码器（生成）
- 自注意力的直觉：每个词根据上下文重新定义自己的含义
- 自注意力对向量的操作：加权求和（线性组合）
- 完整流程：分词 → Embedding → N 层 Transformer Block → 取最后词向量 → 预测下一个 token
- 每一层 = 自注意力 + FFN，堆叠后逐层提炼语义
- 最后一个词为什么信息最全：因果注意力的单向汇聚

### 关键问答
- **为什么每个 token 要和其他 token 做查询？** 不是"绑定"最相似的词，而是决定"我该听谁的话、听多少"。通过加权混合，让每个词的向量融合上下文信息。
- **自注意力对向量做了什么？** 加权求和。新向量 = 所有词的向量按相关度比例混合。
- **为什么最后一个词包含完整上下文？** 因果注意力规则（只能看前面），信息像接力赛从左到右汇聚。
- **每一层是什么？** Transformer Block = 自注意力子层 + FFN 子层，堆叠 N 次逐步提炼语义。

## 阶段 2 ⚠️ — 2026-06-02（复盘待补）

### 核心收获
- "图书馆找书"类比：Q = 我的需求，K = 书的标签，V = 书的内容
- Q、K、V 是同一个输入 X 分别乘以 W_Q、W_K、W_V 得到的三种投影
- W_Q、W_K、W_V 是模型训练出来的固定参数，所有输入共享
- 点积计算匹配度 → Softmax 归一化 → 加权混合 V → 得到新向量
- 点积大 = 向量方向接近 = 相关度高（几何直觉：a·b = |a|·|b|·cosθ）
- 除以 √d_k 的作用：防止 Softmax 梯度消失（缩放数值，避免 one-hot 极端分布）
- d_k = 向量维度，k 来自 Key

### 延伸讨论
- 每层结构相同但参数不同：底层学语法，中层学语义，高层学抽象理解
- 预测只用最后一个词的向量（因果注意力下信息最全）
- 解码过程：最终向量 → 线性层 → 50000 个候选词打分 → Softmax → 选最高分
- 词表里每个词只有一个固定向量，语义区分靠 Transformer 层的上下文加工
- 维度 = 特征数量，4096 维能编码丰富的语义信息
- "苹果"水果 vs 品牌：初始向量相同，经不同上下文的 Transformer 层加工后，最终向量不同
- 信息压缩与丢失：最后一个向量不可能无损包含所有信息，模型学会保留"预测下一个词"最需要的信息
- FFN 的作用：自注意力 = 词与词交流；FFN = 每个词独立消化，内化为更高层抽象
- 因果注意力的掩码机制：处理第 N 个词时只能看前 N-1 个词，通过掩码矩阵（0 保留，-∞ 屏蔽）实现

### 关键公式
```
Q = X · W_Q    （我想问什么）
K = X · W_K    （我的线索是什么）
V = X · W_V    （我的实际内容）

scores = Q · K^T          （点积算匹配度）
scaled = scores / √d_k    （缩放防爆炸）
weights = softmax(scaled)  （变成概率权重）
output = weights · V       （加权混合内容）
```
