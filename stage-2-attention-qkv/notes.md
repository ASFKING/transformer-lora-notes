# 阶段 2：注意力机制的三重奏（Q、K、V）

> 状态：⚠️ 复盘中 | 日期：2026-06-02 | 复盘：2026-06-03

---

## 1. Q、K、V 的直觉

用**图书馆找书**类比：

| 角色 | 含义 | 类比 |
|------|------|------|
| Q（Query） | 我想找什么 | 你的搜索需求 |
| K（Key） | 我能提供什么线索 | 书的标签/关键词 |
| V（Value） | 我的实际内容 | 书里面的实际内容 |

## 2. Q、K、V 的来源

同一个输入 X，分别乘以三个不同的权重矩阵：

```
Q = X · W_Q    （把输入投影到"查询空间"）
K = X · W_K    （把输入投影到"线索空间"）
V = X · W_V    （把输入投影到"内容空间"）
```

- W_Q、W_K、W_V 是**训练出来的固定参数**，所有输入共享
- 每一层有自己独立的 W_Q、W_K、W_V

## 3. 注意力计算的四步流水线

```
Step 1: scores = Q · K^T        ← 点积算匹配度
Step 2: scaled = scores / √d_k  ← 缩放防爆炸
Step 3: weights = softmax(scaled) ← 变成概率权重
Step 4: output = weights · V     ← 加权混合内容
```

### Step 1：点积计算匹配度

Q 的每一行和 K 的每一行做点积，得到一个 n×n 的分数矩阵。

**几何直觉**：
```
a · b = |a| × |b| × cos(θ)

方向一致（θ≈0°）  → cos(θ)≈1  → 点积最大  → "相关"
方向垂直（θ=90°）  → cos(θ)=0  → 点积为零  → "无关"
方向相反（θ=180°） → cos(θ)≈-1 → 点积最负  → "冲突"
```

**类比**：两个人朝同一个方向走，点积最大；互相垂直走，点积为零。

**语义直觉**（待补 Q5）：
- W_Q 把输入投影到"查询空间"——"我在找什么特征"
- W_K 把输入投影到"被查询空间"——"我有什么特征"
- 点积衡量的不是"两个词相不相似"，而是**"一个词的需求和另一个词的供给是否匹配"**
- 这才是 Q/K 分开的本质意义：**需求和供给是两种不同的语义角色**

### Step 2：缩放（除以 √d_k）

- d_k = 向量维度（如 64、128、512）
- 维度越高，点积绝对值天然越大（更多维度累加）
- 大数值扔进 Softmax → 输出接近 one-hot → 梯度消失 → 学不动
- 除以 √d_k 把数值拉回合理范围

**类比**：1000 分制拉回 100 分制，防止一个极端高分碾压其他所有候选。

**数值直觉**：d_k = 64 时，点积典型大小 O(√64) = O(8)，实际可达几十上百。Softmax 对 [100, 1, 1, 1] 输出 ≈ [1, 0, 0, 0]（one-hot），梯度只流过一个位置，其他位置"死了"。

### Step 3：Softmax 归一化

- 把任意实数 → 概率分布（每行加起来 = 1）
- 每一行代表"一个词对所有词的关注度分配"

### Step 4：加权混合 V

- 每个词的新向量 = 所有词的 Value 按关注度权重的混合
- 相关度高的词贡献更多内容
- 4096 维中每一维都是所有词该维的加权混合（调色板类比）

## 4. 完整公式

$$Attention(Q,K,V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

拆解：
| 公式部分 | 含义 |
|:---|:---|
| $QK^T$ | Q 和 K 做点积，算出每对词之间的"相关度分数" |
| $\div \sqrt{d_k}$ | 除以维度的平方根，防止分数太大把 Softmax 撑爆 |
| $softmax(\cdot)$ | 把分数变成概率分布（加起来等于 1 的权重） |
| $\cdot V$ | 用这些权重对 V 做加权求和，得到新向量 |

## 5. 动手实验

见 `attention_demo.py`：用纯 Python 实现了完整的 Self-Attention 计算流程。

句子"我 吃 苹果"，4 维向量：
1. 点积矩阵 → 算出每对词的匹配度
2. 缩放 → 除以 √4 = 2
3. Softmax → 变成概率权重
4. 加权求和 → 得到每个词融合上下文后的新向量

### 实验留疑（Q7 待补）
"吃"的 Q = [0, 1, 0, 1]，"苹果"的 K = [1, 0, 1, 0]，点积 = 0。
- 这意味着：在我们手工选的例子里，"吃"和"苹果"完全不相关
- 但在真实模型里，"吃"和"苹果"应该是强相关的
- **原因**：W_Q、W_K 是训练出来的。我们的例子里手工选了 Q/K 值，不代表模型学到的投影空间
- 模型通过训练学会让"吃"的 Q 和"食物类"的 K 方向对齐

## 6. 延伸知识点

### 每层结构相同但参数不同
- 底层学语法（主谓宾）
- 中层学语义（情感、指代）
- 高层学抽象理解（意图、推理）
- 维度不变（始终 4096），但向量里编码的信息"含金量"逐层提高

### 解码过程
```
最终向量 → 线性层 → 50000 个候选词打分 → Softmax → 选最高分
```
不是"从向量查词"，而是"用向量给所有候选词打分"。

### 词表向量 vs 最终向量
- 词表里每个词只有一个固定向量（Embedding）
- 经过 N 层 Transformer 后，同一个词在不同语境下会变成不同的最终向量
- "苹果"水果 vs 品牌：初始向量相同，加工后不同

### 信息压缩是必然的
- 最后一个向量不可能无损包含所有信息
- 模型学会保留"预测下一个词"最需要的信息，丢弃噪声
- 类比：读完文章写续写，记住的是主题和关键细节，而非每个字

### FFN 的作用
- 自注意力 = 词与词交流（"开会讨论"）
- FFN = 每个词独立消化（"会后独自思考"）
- 把上下文信息内化为更高层抽象

### 因果注意力的掩码机制
- 处理第 N 个词时只能看前 N 个词
- 通过掩码矩阵实现：当前位置标 0（保留），未来位置标 -∞（屏蔽）
- 训练时如果不掩码，模型会"偷看答案"，学不到东西
