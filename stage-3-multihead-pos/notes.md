## 1. 多头注意力（Multi-Head Attention）✅ 已理解

### 核心思想

一个 W_Q 只能学到一种"注意力模式"（比如只关注语法关系）。多头注意力让多个头并行工作，每个头学习不同的注意力模式，最后汇总。

### 类比：多副眼镜 / 多色荧光笔

- 单头 = 只能用一种颜色的荧光笔划重点，只能标记一种模式
- 多头 = 8 种颜色，每种颜色标记不同的关系（语法、语义、局部、长距离……）
- 最后所有颜色叠加，理解更立体

### 数学流程

假设有 h 个头（如 8），模型维度 d_model = 512，每个头维度 d_k = d_model / h = 64：

```
输入 X (512维)
  ├── 头 1：X·W_Q1, X·W_K1, X·W_V1 → Attention → 64维
  ├── 头 2：X·W_Q2, X·W_K2, X·W_V2 → Attention → 64维
  ├── ...
  └── 头 8：X·W_Q8, X·W_K8, X·W_V8 → Attention → 64维

Concat 所有头 → 64×8 = 512维
× W_O → 最终输出 (512维)
```

- 每个头的 Q/K/V 权重矩阵**独立**，各自学到不同的注意力模式
- 最后拼接 + 线性变换（W_O），把多视角信息融合成统一表示

### 不同头学到的模式（示例）

| 头 | 关注的模式 | 例子（"小明 送 小红 一朵 花"） |
|:---:|:---|:---|
| 头 1 | 语法依赖（谁对谁做了什么） | "花" → 关注"送"（动宾关系） |
| 头 2 | 局部上下文（附近的词） | "花" → 关注"一朵"（紧邻量词） |
| 头 3 | 长距离指代（远距离语义关联） | "花" → 关注"小红"（接收者） |
| 头 4 | 语义角色（施事/受事） | "花" → 关注"小明"（发起者） |

### 关键理解

- 多头的价值 ≠ 保留维度信息（单头 512 维和多头 8×64 维输出维度相同）
- 多头的价值 = **每个头专注学一种注意力模式，互不干扰**（模型容量问题）
- 单头被迫把所有模式混在一起学，哪个都学不精

## 2. 位置编码（Positional Encoding）🔄 需要继续讲解

### 为什么需要位置编码？

**问题**：注意力计算公式 $\text{Attention}(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$ 中，没有任何地方标记"这是第几个词"。

- 每个词的 Q、K、V 由输入 X 乘以 W_Q、W_K、W_V 得到
- 同一个词不管在什么位置，Q、K、V 都相同
- 所以"我吃苹果"和"苹果吃我"会被视为相同的输入（bag of words）
- 语言是有序的，分不清顺序就学不到语法

### 绝对位置编码（原始 Transformer）

用正弦/余弦函数为每个位置生成唯一向量，加到词向量上：

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

- 每个位置有独特编码（像每层楼有不同气味）
- 位置间的关系可通过线性变换表达
- 加法融合：每维同时编码语义 + 位置，模型自动学会区分

**局限**：
- 编码的是绝对位置，泛化性差（训练 512 长度，推理时第 513 个词没见过）
- 忽略了相对距离（语言理解中"离动词多远"比"在第几个位置"更重要）

### RoPE（旋转位置编码）🔄 待补充

- 核心思想：通过旋转 Q 和 K 来编码位置，点积结果只取决于相对距离
- 三大优势：天然编码相对位置、更好的外推性、无额外参数
- 为什么 GPT-4 能处理长文本：RoPE + 外推技术
