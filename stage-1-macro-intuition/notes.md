# 阶段 1：Transformer 的宏观直觉

> 状态：✅ 已完成 | 日期：2026-06-02

---

## 1. RNN 的痛点

RNN（循环神经网络）像**排队传话**：每个词必须等前一个词处理完才能开始。

- **串行处理**：无法并行，训练慢
- **长距离遗忘**：信息在传递过程中逐步衰减，句子太长就记不住开头

## 2. Transformer 的核心突破

Transformer 像**圆桌会议**：所有词同时互相看，不需要排队。

- **全局扫描**：每个词可以同时关注所有其他词
- **并行计算**：训练效率大幅提升
- **长距离依赖**：无论两个词距离多远，直接建立连接

## 3. Encoder-Decoder 架构

| 组件 | 角色 | 类比 |
|------|------|------|
| Encoder（编码器） | 理解输入 | 听得懂的耳朵 |
| Decoder（解码器） | 生成输出 | 能说的嘴 |

典型应用：机器翻译（中文输入 → Encoder 理解 → Decoder 生成英文）

## 4. 自注意力（Self-Attention）

**直觉**：每个词根据上下文重新定义自己的含义。

> "苹果"在"我吃苹果"和"苹果发布新手机"中的含义不同，
> 自注意力机制让模型根据周围词来调整每个词的向量表示。

**操作**：加权求和（线性组合）

- 新向量 = 所有词的向量 × 相关度权重 的总和
- 相关度高的词权重更大，贡献更多

## 5. 完整处理流程

```
输入文本
  ↓
分词（Tokenization）
  ↓
Embedding（每个词 → 向量）
  ↓
N 层 Transformer Block（每层 = 自注意力 + FFN）
  ↓
取最后一个词的向量（因果注意力下信息最全）
  ↓
线性层 → 词表打分 → Softmax → 预测下一个 token
```

## 6. 为什么最后一个词信息最全？

因果注意力规则：每个词只能看到它**前面**的词。

信息从左到右像接力赛一样汇聚：
- 第 1 个词：只知道自己
- 第 2 个词：知道 1 和 2
- 第 3 个词：知道 1、2、3
- 最后一个词：知道所有词

所以取最后一个词的向量来做预测。

## 7. 每一层 = 自注意力 + FFN

| 子层 | 作用 | 类比 |
|------|------|------|
| 自注意力子层 | 词与词之间交流信息 | 开会讨论 |
| FFN 子层 | 每个词独立消化信息 | 会后独自思考 |

堆叠 N 层，逐层提炼语义：
- 底层：学语法（主谓宾）
- 中层：学语义（情感、指代）
- 高层：学抽象理解（意图、推理）

## 关键问答

**Q: 为什么每个 token 要和其他 token 做查询？**
A: 不是"绑定"最相似的词，而是决定"我该听谁的话、听多少"。通过加权混合，让每个词的向量融合上下文信息。

**Q: 自注意力对向量做了什么处理？**
A: 加权求和。新向量 = 所有词的向量按相关度比例混合。不同上下文 → 不同权重 → 不同的新向量。

**Q: 为什么最后一个词包含完整上下文？**
A: 因果注意力规则（只能看前面），信息像接力赛一样从左到右汇聚，最后一个词收到的是所有前序信息的总和。

**Q: 每一层是什么？**
A: 就是隐藏层。Transformer 的每一层 = 一个 Transformer Block（自注意力子层 + FFN 子层），堆叠 N 次逐步提炼语义。
